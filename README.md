# Shanghai Heat Risk Analysis (HRI)

Automated GIS pipeline for neighbourhood-scale heat-risk assessment and heat-shelter supply-demand mismatch analysis in Shanghai, using the **HRI + Shelter** framework.

## What this project does

Instead of producing a generic "hot-zone" map, this pipeline identifies **where heat risk is high AND cooling resources are insufficient** — a spatial equity analysis.

```
Priority = Heat Risk Index (HRI)  −  Shelter Provision Index
```

The pipeline outputs two actionable priority maps:

- **OHSPI** (Outdoor Heat Shelter Priority) — blocks that are hot + lack green space
- **IHSPI** (Indoor Heat Shelter Priority) — blocks that are hot + lack air-conditioned public facilities

## Pipeline overview

| Step | Script | Description |
|------|--------|-------------|
| 0 | `00_setup.py` | Install dependencies, create directories |
| 1 | `01_download_data.py` | Download UTCI, nightlight, GDP rasters; synthesize population proxy from OSM buildings |
| 2 | `02_preprocess.py` | Clip and reproject all rasters to Shanghai extent (EPSG:32651) |
| 3 | `03_create_blocks.py` | Generate road-enclosed urban blocks via `polygonize` + 500m fishnet infill |
| 4 | `04_zonal_stats.py` | Compute per-block statistics: UTCI, population, nightlight, GDP, green area, POI density |
| 5 | `05_calculate_hri.py` | Calculate HRI = Hazard x Exposure x Vulnerability |
| 6 | `06_calculate_shelters.py` | Calculate OHSI (green space per capita) and IHSI (weighted POI density per capita) |
| 7 | `07_priority_index.py` | Calculate OHSPI, IHSPI, and apply 7-class Jenks classification |
| 8 | `08_visualize.py` | Generate 5 thematic maps (300 dpi PNG) |
| 9 | `09_generate_qgis_project.py` | Generate `.qgz` project file with all layers pre-styled |

## Quick start

### Prerequisites

- [Anaconda](https://www.anaconda.com/) or [Miniconda](https://docs.conda.io/en/latest/miniconda.html)
- ~2 GB free disk space for downloaded data

### Setup

```bash
conda create -n hri python=3.12 geopandas rasterio rasterstats numpy pandas \
  matplotlib contextily mapclassify shapely pyproj fiona requests scipy \
  matplotlib-scalebar -c conda-forge -y

conda activate hri
```

### Run

```bash
cd "QGIS Oxford"
python scripts/run_all.py
```

Or run individual steps:

```bash
python scripts/05_calculate_hri.py
```

### View results in QGIS

Double-click `output/shanghai_heat_risk.qgz` to open the pre-configured QGIS project with:

- 5 analysis layers (HRI, OHSI, IHSI, OHSPI, IHSPI) with graduated colour renderers
- 4 raster layers (UTCI, Population, Nightlight, GDP) with pseudocolour symbology
- 6 OSM context layers (roads, water, landuse, buildings, POIs, transport)
- CartoDB Positron and OpenStreetMap basemaps

## Model

### HRI = Hazard x Exposure x Vulnerability

| Component | Source | Normalization |
|-----------|--------|---------------|
| **Hazard** | UTCI August 2022 (GloUTCI-M, ~1km) | Positive [0.1, 0.9] |
| **Exposure** | Population density (OSM buildings proxy) | Positive |
| **Vulnerability** | Mean of: Nightlight (negative), GDP (negative), Population density (positive) | Mixed |

### Shelter indices

| Index | Formula |
|-------|---------|
| **OHSI** | normalize(green space area / population) |
| **IHSI** | normalize(weighted POI density / population density) |
| **OHSPI** | HRI_norm - OHSI |
| **IHSPI** | HRI_norm - IHSI |

POI categories and operational time weights:

| Category | Examples | Weight (fraction of 24h open) |
|----------|----------|-------------------------------|
| Commercial | mall, supermarket | 0.50 |
| Restaurant | cafe, fast_food, bakery | 0.625 |
| Cultural | library, museum, cinema | 0.33 |
| Transport | railway_station (metro) | 0.71 |

## Spatial units

Road-enclosed urban blocks generated by polygonizing the OSM road network. City-centre blocks are naturally small (~200-500m); suburban blocks are larger. Areas not covered by the road network are filled with a 500m fishnet grid.

**24,214 blocks** covering the full Shanghai administrative area.

## Data sources

| Dataset | Resolution | Source |
|---------|-----------|--------|
| UTCI August 2022 | ~1 km | [GloUTCI-M (Zenodo)](https://zenodo.org/records/8310513) |
| Nightlight 2021 | ~500 m | [PCNL (Zenodo)](https://zenodo.org/records/7612389) |
| GDP 2020 | ~1 km | [Gridded GDP 30-arcsec (Zenodo)](https://zenodo.org/records/13943886) |
| Population proxy | ~100 m | Synthesized from OSM building footprints (24.9M total) |
| Roads, POIs, green space | Vector | [OpenStreetMap via Geofabrik](https://download.geofabrik.de/asia/china/shanghai.html) |

## Notes

- This is a **method demonstration** using historical data, not a real-time system
- No LCZ classification is used; the framework is human-centric (UTCI-based)
- The core insight is the **gap** (HRI - Shelter), not heat or resources alone
- All normalization uses [0.1, 0.9] range to prevent zero-multiplication in HRI
